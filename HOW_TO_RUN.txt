What I Built
━━━━━━━━━━━━
SignBridge is a Chrome Extension + Python Backend system that:
  • Detects American Sign Language (ASL) from your webcam in real-time
  • Converts signs to spoken words using Google Gemini AI
  • Speaks the translated text aloud in Google Meet calls
  • Shows a live hand-avatar panel animating what others are saying
  • Records meeting transcripts and generates AI PDF summaries
  • Works ONLY on https://meet.google.com

STEP 1 — PYTHON REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
You MUST use Python 3.10 or 3.11 (TensorFlow 2.13 is not compatible
with Python 3.12+).

Check your Python version:
  python --version

If you have 3.12+, install Python 3.11 from:
  https://www.python.org/downloads/release/python-3119/

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 2 — SETUP BACKEND
━━━━━━━━━━━━━━━━━━━━━━

Open a terminal/command prompt and run:

  cd "C:\Users\Arokiya Nithish\OneDrive\Pictures\LLM\signbridge\backend"

Create virtual environment:
  python -m venv venv
  venv\Scripts\activate         (Windows)
  (or: source venv/bin/activate on Mac/Linux)

Install all dependencies:
  pip install -r requirements.txt

This installs: FastAPI, TensorFlow, MediaPipe, OpenCV,
Google Gemini, ReportLab, SQLAlchemy, and more.

STEP 4 — CONFIGURE ENVIRONMENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

In the backend folder, copy the example .env file:
  copy .env.example .env

Open .env in Notepad and fill in:
  GEMINI_API_KEY=paste_your_key_here
  JWT_SECRET_KEY=any_random_long_string_here

Generate a JWT secret key with Python:
  python -c "import secrets; print(secrets.token_hex(32))"

STEP 8 — USE IN GOOGLE MEET
━━━━━━━━━━━━━━━━━━━━━━━━━━━

6. Start signing → detected signs appear in the panel

STEP 9 — TRAIN THE AI MODEL (Optional but Recommended)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Download ASL Dataset from Kaggle:
  1. Go to: https://kaggle.com/datasets/grassknoted/asl-alphabet
  2. Download and extract it to: backend/data/
  3. You will see two folders: `asl_alphabet_train` and `asl_alphabet_test`.
     - You ONLY need `asl_alphabet_train`.
     - The training script will automatically split `asl_alphabet_train` into
       80% training and 20% validation.
  4. Ensure your folder structure looks exactly like this:
     backend/data/asl_alphabet/asl_alphabet_train/A/
     backend/data/asl_alphabet/asl_alphabet_train/B/
     ... etc ...

Run training:
  cd backend
  python sign_recognition/train.py

  This takes 2-4 hours on a GPU.
  Best model saved to: backend/models/sign_model.h5

Without a trained model, the backend runs in DEMO MODE
(simulates predictions so you can test the full system).

TROUBLESHOOTING
━━━━━━━━━━━━━━━

Problem: "Cannot reach backend"
Fix: Make sure backend server is running (Step 5)

Problem: Extension not showing in Meet
Fix: Be in an active meeting (not just the lobby)
     Reload the Meet tab after loading extension

Problem: Camera access denied
Fix: Chrome Settings → Privacy → Camera → Allow for meet.google.com

Problem: TensorFlow install fails
Fix: Use Python 3.10 or 3.11 (NOT 3.12+)
     Or use Google Colab for training

Problem: "Model not found" error
Fix: Run training script or place sign_model.h5 in backend/models/
     Server runs in demo mode without the model
